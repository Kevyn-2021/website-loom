# Writing 内容维护说明

这份文档是 KevinZ 网站写作模块的唯一维护说明。以后新增 Newsletter、调整文章格式、添加文内图片，都以这里为准。

## 1. 文件放在哪里

所有写作文章放在：

```text
loom/content/writing/
```

每篇文章一个 Markdown 文件。

## 2. 文件命名

文章文件名采用：

```text
YYYY-MM-DD_文章标题.md
```

示例：

```text
2026-06-20_定价：既不是数学题，也不是语文题.md
```

网站会按照文件名或 frontmatter 里的 `date` 识别文章日期，并按日期倒序排列，最新文章在最前。这里的日期表示文章原始写作日期，不表示上传到网站的日期。

不符合这个命名规则的 Markdown 文件不会显示在写作列表里。因此，本说明文档不会出现在前台 Writing 页面。

## 3. 文章开头格式

每篇文章开头必须写 frontmatter：

```md
---
title: 文章标题
date: 2026-06-27
tags: [Product]
description: 这一句用于写作列表页，不会重复进入正文。
---
```

字段说明：

- `title`：文章标题，会显示在列表页和详情页；
- `date`：文章日期，格式为 `YYYY-MM-DD`；
- `tags`：文章标签，只能使用固定标签；
- `description`：一句话摘要，用于列表页和详情页标题区，不会进入正文。

如果没有写 `description`，网站会自动提取正文第一段作为列表描述。但为了让列表页更稳定，建议手动写 `description`。

## 4. Tag 规则

Tag 有且仅有以下四类：

```text
Go-to-Market
Product
AI
Career
```

可以单选，也可以多选：

```md
tags: [Go-to-Market, Product]
```

不要新增近义标签，例如 `GTM`、`Marketing`、`产品`、`职业`。如果未来确实需要新增标签，应先修改网站规则和这份说明文档。

## 5. 列表页展示什么

Writing 列表页展示：

- 时间；
- 标题；
- 一句话描述；
- Tag。

列表页按照文章日期倒序排列。

## 6. 详情页展示什么

点击文章卡片后进入详情页。

详情页展示：

- 返回写作入口；
- 时间；
- 标题；
- 一句话描述；
- Tag；
- 正文内容。

`description` 只作为摘要显示，不会重复进入正文。

## 7. 正文写法

正文从 frontmatter 之后开始。

支持常用 Markdown 写法：

```md
## 二级标题

正文段落。

**加粗文字**

- 列表项一
- 列表项二

---
```

正文段落之间用空行分隔。

## 8. 添加图片

图片文件建议放在：

```text
loom/public/assets/writing/
```

示例：

```text
loom/public/assets/writing/price-waterfall.png
```

在文章 Markdown 中这样引用：

```md
![价格瀑布示意图](/assets/writing/price-waterfall.png)
```

语法说明：

- `![价格瀑布示意图]` 是图片说明，也会作为图片的无障碍描述；
- `(/assets/writing/price-waterfall.png)` 是网站访问路径；
- 路径必须从 `/assets/` 开始，不要写本机绝对路径。

图片会在文章详情页显示，不会在写作列表页显示。
